# Blogging Skill

## Workflow

```
Outline → Draft → Edit → SEO → Publish → Promote
```

Each step requires approval before proceeding to the next.

## Step 1: Outline

1. Confirm topic, audience, and tone with user
2. Research the topic if needed (invoke research skill)
3. Create structured outline:
   - **Hook** — Opening that grabs attention (question, stat, story)
   - **Context** — Why this matters now
   - **Body** — 3-5 key sections with subpoints
   - **Conclusion** — Summary + call to action
4. Present outline for approval

## Step 2: Draft

Write the full post following the approved outline:

- **Length**: 800-2000 words (default ~1200)
- **Paragraphs**: 2-4 sentences max
- **Headers**: H2 for sections, H3 for subsections
- **Voice**: Conversational but authoritative
- **Format**: Markdown

### Structure Guidelines

```markdown
# [Title — under 60 chars for SEO]

[Hook paragraph — make them want to keep reading]

[Context — why this matters]

## [Section 1]
[Content with examples]

## [Section 2]
[Content with data or stories]

## [Section 3]
[Content with actionable advice]

## Conclusion
[Summary of key points]
[Call to action — what should the reader do next?]
```

## Step 3: Edit

Review for:
- Clarity and flow
- Redundancy removal
- Active voice preference
- Jargon elimination (unless audience expects it)
- Grammar and punctuation

## Step 4: SEO Optimization

- Title under 60 characters
- Meta description under 155 characters
- Target keyword in title, first paragraph, and 2-3 headers
- Internal/external links where natural
- Alt text for any images

## Step 5: Publish

Publish via the available CMS or output as markdown for manual publishing.

## Step 6: Promote (Optional)

Generate social media posts:
- **Twitter/X**: Hook + key insight + link (under 280 chars)
- **LinkedIn**: Professional angle + 3 key takeaways + link
- **Bluesky**: Conversational hook + link

## Rules

- Always get topic approval before drafting
- Never publish without explicit user confirmation
- Include at least one concrete example per section
- Avoid clickbait — deliver on the title's promise
