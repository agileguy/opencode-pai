# Thinking Skill

## Mode Selection

| Mode | Trigger | Purpose |
|------|---------|---------|
| **First Principles** | "first principles", "from scratch", "fundamentally" | Decompose to base truths, rebuild |
| **Iterative Analysis** | "analyze", "think about", "what do you think" | Progressive depth exploration |
| **Brainstorm** | "brainstorm", "ideas", "creative", "possibilities" | Diverge widely, then converge |
| **Red Team** | "red team", "devil's advocate", "argue against" | Systematic adversarial critique |
| **Debate** | "debate", "pros and cons", "both sides" | Multi-perspective structured argument |

## First Principles Mode

1. **State the problem** clearly
2. **List assumptions** everyone takes for granted
3. **Challenge each assumption** — Is it actually true? What evidence?
4. **Identify base truths** — What remains after stripping assumptions?
5. **Rebuild from fundamentals** — What solution emerges from base truths?
6. **Compare** — How does the first-principles solution differ from the conventional one?

## Iterative Analysis Mode

Layer depth progressively:

1. **Surface**: What does this look like at face value?
2. **Structure**: What patterns, systems, or forces are at work?
3. **Root Cause**: Why do those patterns exist?
4. **Implications**: What follows if this analysis is correct?
5. **Action**: What should be done given this understanding?

## Brainstorm Mode

1. **Diverge** — Generate 10-20 ideas without judgment
   - Wild ideas welcome
   - Build on previous ideas
   - Quantity over quality in this phase
2. **Cluster** — Group related ideas into themes
3. **Evaluate** — Score each cluster on feasibility and impact
4. **Converge** — Select top 3 ideas, flesh them out
5. **Present** — Structured proposal for best option(s)

## Red Team Mode

Systematically argue against the proposal:

1. **Steel-man** the proposal first (show you understand it)
2. **Attack assumptions** — What must be true for this to work?
3. **Find failure modes** — How could this go wrong?
4. **Identify blind spots** — What hasn't been considered?
5. **Assess risks** — Likelihood and impact of each failure
6. **Verdict** — Overall assessment with specific concerns

## Debate Mode

1. **Frame the question** precisely
2. **Argue FOR** — Strongest possible case
3. **Argue AGAINST** — Strongest possible case
4. **Synthesize** — Where do the sides actually agree/disagree?
5. **Recommend** — Given the full picture, what's the best path?

## Output Format

Always structure output with:
- Clear mode label at the top
- Numbered steps matching the mode
- Bold key insights
- Explicit conclusion or recommendation

## Rules

- Always identify which mode is being used
- Be genuinely rigorous — don't softball analysis
- Distinguish facts from opinions clearly
- Acknowledge uncertainty where it exists
- Provide actionable conclusions, not just analysis
