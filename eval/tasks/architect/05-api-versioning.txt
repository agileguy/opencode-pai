Design an API versioning strategy for a REST API with 30 endpoints serving 500 active clients. The API needs to introduce breaking changes (removing fields, changing response shapes) while maintaining backward compatibility for at least 6 months. Write your design to the output directory as api-versioning.md. Evaluate: URL path versioning (/v1/, /v2/), header-based versioning (Accept-Version), and query parameter versioning (?version=2). Include: migration path for clients, deprecation communication strategy, implementation complexity, testing strategy, and your recommendation with justification.
