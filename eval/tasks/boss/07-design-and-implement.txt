Build a TypeScript key-value store with TTL (time-to-live) support. First, delegate a design document to an architect agent covering: data structure choice, expiration strategy (lazy vs eager vs hybrid), and memory management. Then, delegate the implementation with tests to an engineer agent. The store should support: set(key, value, ttlMs?), get(key), delete(key), has(key), and clear(). Expired entries should not be returned by get().
